단순한 자연어 채팅을 넘어, 슬래시(/)로 시작하는 내장 명령어를 통해 터미널 내에서 개발 워크플로우 자체를 즉시 통제합니다.

💡 퇴근 시간을 앞당기는 자동화 처리 사례

  • ✓ 테스트되지 않은 레거시 코드에 대한 단위 테스트 자동 작성
  • ✓ 프로젝트 전체에 산재한 까다로운 린트(Lint) 오류 일괄 수정
  • ✓ 깃(Git) 환경에서의 복잡한 병합 충돌 해결 및 종속성 업데이트

Skills (스킬)

1 상태의 분리와 외부화

Stateless Agent Design

프롬프트 내에 비즈니스 상태를 우겨넣지 마세요. 에이전트는 일회성(Ephemeral) 워커로 설계하고, 기억해야 할 상태 데이터는 반드시 외부 DB로 이관해야 합니다.

2 작고 집중된 에이전트

Small, Focused Agents

한 에이전트가 20단계를 처리하면 고장 납니다. 최대 3~10단계의 명확한 목표만 처리하도록 작게 쪼개는 것이 높은 신뢰성을 보장합니다.

3 컨텍스트 창의 명시적 관리

Manage Context Windows Explicitly

과거 대화 기록을 맹목적으로 이어 붙이지 마십시오. 워커에게는 작업을 수행하는 데 필요한 '최소한의 정보'만 쥐여주는 것이 핵심입니다.

4 엄격한 오케스트레이션

Strict Orchestration

하위 에이전트들에게 복잡한 제어 흐름(Control Flow)을 맡기지 마세요. 조율과 예외 처리는 최상단의 오케스트레이터가 전담해야 합니다.
